What is Machine Learning?

Machine Learning (ML) is a branch of Artificial Intelligence that enables computer systems to learn patterns from data and make predictions or decisions without being explicitly programmed for every rule.

Key idea

Instead of hard‚Äëcoding rules, we train a model on examples so it can generalize to new, unseen inputs.

Data ‚Üí Model ‚Üí Prediction

Goal

Minimize error on future data by learning parameters from past data. This is measured using a loss function.

Generalization

Ingredients

Data, features/representations, a learning algorithm, an objective (loss), and evaluation metrics.

Accuracy ‚Ä¢ F1 ‚Ä¢ RMSE
Simple definition: ML finds relationships in data and uses them to make useful predictions or decisions automatically.

Typical ML Workflow

1) Define the problem

What is the task? (e.g., predict price, classify email as spam/not‚Äëspam)

2) Collect & prepare data

Gather, clean, split data (train/validation/test). Engineer features if needed.

3) Choose a model

Select algorithms suitable for the task and data size/quality.

4) Train

Fit the model on training data by minimizing loss (e.g., gradient descent).

5) Evaluate

Assess with appropriate metrics; diagnose overfitting/underfitting.

6) Deploy & monitor

Serve the model and track performance drift over time.

Types of Machine Learning

1) Supervised Learning

Learn from labeled data (input ‚Üí known target). Used for prediction and classification.

Tasks

  • Regression: predict continuous values (e.g., house price)
  • Classification: predict categories (e.g., spam vs. not spam)

Common algorithms

  • Linear/Logistic Regression
  • Decision Trees & Random Forests
  • Support Vector Machines (SVM)
  • k‚ÄëNearest Neighbors (kNN)
  • Neural Networks

Examples

  • Credit risk scoring
  • Image recognition (cats/dogs)
  • Demand forecasting

2) Unsupervised Learning

Learn patterns from unlabeled data. There is no target variable to predict.

Tasks

  • Clustering (k‚Äëmeans, DBSCAN, hierarchical)
  • Dimensionality Reduction (PCA, t‚ÄëSNE, UMAP)
  • Association Mining (market baskets)

Use cases

  • Customer segmentation
  • Anomaly detection
  • Feature extraction/visualization

Notes

  • No single ‚Äúaccuracy‚Äù ‚Äî evaluate by cohesion/separation or downstream utility.

3) Semi‚Äësupervised Learning

Combine a small set of labeled data with a large pool of unlabeled data to improve performance.

When to use

  • Labeling is expensive (e.g., medical images)
  • Plenty of raw data available

Techniques

  • Label propagation / self‚Äëtraining
  • Consistency regularization
  • Pseudo‚Äëlabeling

Benefit

  • Approaches supervised accuracy with fewer labels

4) Self‚Äësupervised Learning

Learn representations by creating supervision from the data itself (e.g., masking parts of input and predicting them).

Examples

  • Masked language modeling (text)
  • Contrastive learning (images/audio)
  • Autoencoders

Why it matters

  • Pretraining on large unlabeled datasets
  • Improves downstream tasks with few labels

Outcome

  • Rich embeddings/features for transfer learning

5) Reinforcement Learning (RL)

An agent learns to act in an environment by taking actions to maximize cumulative reward.

Core concepts

  • State, action, reward, policy, value
  • Exploration vs. exploitation

Algorithms

  • Q‚Äëlearning, SARSA
  • Deep Q‚ÄëNetworks (DQN)
  • Policy Gradient, PPO

Applications

  • Game playing (e.g., Go, Atari)
  • Robotics & control
  • Recommendation strategies
Deep Learning in one minute

Deep Learning is a subset of ML using multi‚Äëlayer neural networks that learn hierarchical representations. It shines with large datasets and unstructured data (images, audio, text).

Quick FAQ

What is the difference between AI, ML, and DL?

AI is the broad goal of making machines smart. ML is a subset focused on learning from data. Deep Learning (DL) is a subset of ML using deep neural networks.

How do we measure success?

Use appropriate metrics: Accuracy/F1 for classification, MAE/RMSE/R¬≤ for regression, reward for RL. Always validate on data not used for training.

What are common pitfalls?

Data leakage, overfitting, biased data, poor evaluation splits, and deploying without monitoring for drift.
